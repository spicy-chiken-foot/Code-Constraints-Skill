# 代码约束 Skill 使用说明

## 简介

该 Skill 用于约束 AI 在编写、修改、重构或审查代码时的行为。

它要求 AI 严格遵守最小改动原则，只实现明确需求，优先复用项目已有代码，避免重复造轮子、过度封装、擅自引入依赖和无关重构，确保生成代码精简、聚焦，并符合项目现有代码风格。

---

## 适用场景

当使用 AI 工具处理以下任务时，建议启用该 Skill：

- 编写新功能
- 修改已有代码
- 修复 Bug
- 代码重构
- 代码审查
- 补全代码
- 生成测试代码
- 优化局部逻辑

---

## 核心目标

该 Skill 的核心目标是让 AI：

1. 不超出需求边界
2. 不做无关改动
3. 不重复造轮子
4. 不过度封装
5. 不擅自引入新依赖
6. 不进行无关重构
7. 保持代码精简
8. 遵守项目现有风格

---

## 十条核心约束

1. 只实现明确需求
2. 保持最小改动范围
3. 优先复用已有代码
4. 不擅自引入新依赖
5. 避免过度封装和过度抽象
6. 控制代码粒度
7. 不做无关重构和顺手优化
8. 遵守项目现有风格
9. 不改变已有公共 API 行为
10. 输出前必须自检
